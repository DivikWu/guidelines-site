## 规范之间的关系

```
颜色系统 ─┬─► 组件视觉
字体排版 ─┤
          │
间距 ─────┼─► 组件布局 ─► 页面模式
布局 ─────┤
          │
圆角 ─────┼─► 组件形状
层级与阴影┤
          │
图标 ─────┴─► 组件元素
          
动效 ─────────► 交互反馈

无障碍 ───┬─► 贯穿所有规范
国际化 ───┘
```

### 视觉基础

**颜色系统** 与 **字体排版** 共同构成品牌视觉识别的核心，是所有组件的视觉起点。

### 空间系统

**间距** 定义元素之间的距离关系，**布局** 定义页面的整体结构。两者结合确保界面的节奏感与一致性。

### 形状与层级

**圆角** 与 **层级/阴影** 共同定义组件的三维感知，影响用户对元素重要性与交互状态的理解。

### 贯穿性规范

**无障碍** 与 **国际化** 不是独立的模块，而是需要在其他所有规范的应用中持续考虑的原则。

## 使用建议

1. **先理解后使用**：通读各规范，理解设计意图，而非机械套用
2. **Token 优先**：使用设计 Token（颜色、间距、圆角等）而非硬编码值
3. **保持一致**：遇到规范未覆盖的场景，优先复用已有模式
4. **及时反馈**：发现规范不足或问题，反馈至设计系统团队
