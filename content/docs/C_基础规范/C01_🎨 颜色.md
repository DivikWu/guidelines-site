# 🎨 颜色系统

颜色是品牌识别与用户体验的核心。YAMI 设计系统通过语义化的颜色 Token 体系，确保在不同平台与场景中保持视觉一致性与可访问性。 测试同步

---

## 品牌色

| Token | 色值 | 用途 |
|-------|------|------|
| brand/primary | `#FF0000` | 品牌标识、Logo、广告主视觉 |
| ui/primary | `#E00000` | 主按钮、通知标签、状态徽标、品牌高亮 |

### 品牌主色 Brand Primary

用于品牌标识和对外传播场景，如 Logo、广告主视觉等，突出品牌识别性。

> 不建议在产品界面中直接用于正文文字或大面积背景，以避免影响可读性与用户体验。

### UI 主色 UI Primary

作为界面主操作色，应用于主按钮、通知标签、状态徽标等交互类组件，以及界面中的品牌高亮区域。

> UI Primary (#E00000) 在 Brand Primary (#FF0000) 的基础上进行亮度微调，满足 WCAG AA 对比度要求（≥ 4.5:1），在强化视觉识别的同时兼顾可访问性。

---

## 语义色

语义色用于传达特定含义或状态，确保用户在不同场景下能够快速理解信息。

### 文字色

| Token | 色值（亮色模式） | 透明度 | 用途 |
|-------|------------------|--------|------|
| text/primary | `rgba(0,0,0,0.87)` | 87% | 主要文字，正文标题 |
| text/secondary | `rgba(0,0,0,0.55)` | 55% | 次要文字，辅助说明 |
| text/disabled | `rgba(0,0,0,0.29)` | 29% | 禁用态文字 |

### 边框与分割线

| Token | 色值（亮色模式） | 透明度 | 用途 |
|-------|------------------|--------|------|
| border/primary | `rgba(0,0,0,0.08)` | 8% | 主要边框，容器边界 |
| divider/normal | `rgba(0,0,0,0.08)` | 8% | 分割线，区块分隔 |

### 背景与表面

| Token | 色值（亮色模式） | 用途 |
|-------|------------------|------|
| background/primary | `#FFFFFF` | 页面主背景 |
| surface/primary | `#FFFFFF` | 卡片、弹窗等表面 |
| fill/default | `#FFFFFF` | 默认填充色 |

### 辅助色板

| Token | 色值 | 用途 |
|-------|------|------|
| blue/50 | `#F0F3FA` | 浅蓝背景，信息类提示 |
| sky/50 | `#EEF6FE` | 浅天蓝背景，预览区域 |



### 功能色

功能色用于传达系统反馈状态。

| 类型 | 用途 |
|------|------|
| Success | 成功、完成、正向反馈 |
| Warning | 警告、注意、需关注 |
| Error | 错误、失败、危险操作 |
| Info | 提示、说明性信息 |



## 调色板

设计系统包含完整的调色板体系，每个颜色家族包含从浅到深的色阶（如 50、100、200...900），用于：

- 不同状态的视觉区分（hover、active、disabled）
- 层级表达与视觉深度
- 暗色模式适配

> [!note] 使用建议
> 日常设计中优先使用语义化 Token（如 `text/primary`），而非直接使用色阶值，以便于主题切换与后续维护。

---




## 可访问性

- 前景与背景的对比度需满足 WCAG AA 标准（正文 ≥ 4.5:1，大文本 ≥ 3:1）
- 不依赖颜色作为唯一信息传达方式，需结合图标或文字说明
- 功能色需在色盲模式下可区分
