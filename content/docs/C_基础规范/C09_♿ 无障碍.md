# ♿ 无障碍

无障碍设计确保所有用户（包括视觉、听觉、运动、认知障碍用户）都能平等地使用产品。YAMI 设计系统将无障碍作为核心原则贯穿设计与开发流程。

---

## 基本原则

遵循 WCAG 2.1 指南的四大原则：

### 可感知（Perceivable）

- 提供文本替代方案（图片 alt、图标 aria-label）
- 支持屏幕阅读器
- 不仅依赖颜色传达信息

### 可操作（Operable）

- 支持键盘导航
- 提供足够的点击区域
- 避免闪烁内容

### 可理解（Understandable）

- 使用清晰的语言
- 保持交互的一致性
- 提供错误提示与帮助

### 健壮（Robust）

- 使用语义化 HTML
- 兼容辅助技术
- 持续测试与迭代

## 视觉无障碍

### 颜色对比度

| 内容类型 | 最低对比度 |
|----------|------------|
| 正文文本 | 4.5:1（AA） |
| 大文本（≥18px 或 ≥14px 加粗） | 3:1（AA） |
| 图形与 UI 组件 | 3:1（AA） |

### 不依赖颜色

- 链接使用下划线或其他视觉提示
- 错误状态结合图标与文字
- 图表使用图案或标签补充颜色

### 文字可缩放

- 支持浏览器/系统字体缩放
- 200% 缩放下布局不破坏

---

## 交互无障碍

### 键盘导航

- 所有交互元素可通过 Tab 键访问
- 提供清晰的焦点指示器
- 逻辑的 Tab 顺序

### 焦点管理

- 弹窗打开时焦点移入
- 弹窗关闭时焦点返回触发元素
- 焦点陷阱用于模态场景

### 点击区域

- 最小点击区域建议 ≥ 44×44px
- 相邻可点击元素间保留间距

---

## 内容无障碍

### 替代文本

- 信息性图片提供描述性 alt
- 装饰性图片使用空 alt 或 aria-hidden
- 复杂图表提供文本摘要

### 表单

- 输入框关联 label
- 错误信息与输入框关联
- 提供清晰的必填项标识

### 多媒体

- 视频提供字幕
- 音频提供文字稿
- 自动播放可控制

1. **自动化工具**：使用 axe、Lighthouse 等工具扫描
2. **键盘测试**：仅用键盘完成核心流程

4. **真实用户测试**：邀请障碍用户参与测试

## 常见问题

**Q：无障碍会影响视觉设计吗？**
A：良好的无障碍设计对所有用户都有益。高对比度、清晰的结构同样提升普通用户体验。

**Q：如何平衡品牌色与对比度要求？**
A：可调整品牌色的使用方式（如用于背景而非文字），或微调亮度以满足对比度。
