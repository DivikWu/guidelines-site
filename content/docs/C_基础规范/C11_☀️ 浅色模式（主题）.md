# ☀️ 浅色模式

本文档整理 YAMI 设计系统当前**浅色模式**的完整色值、背景层级及语义变量，供设计与开发查阅。

---

## 文档说明

### 数据来源

- **Token 源**：[tokens/tokens.json](tokens/tokens.json)，经 `scripts/generate-tokens.py` 生成 [styles/tokens.css](styles/tokens.css)。
- **语义变量**：[app/globals.css](app/globals.css) 中 `:root`（浅色）定义 `--foreground-*`、`--background-*`、`--state-*` 等；[styles/theme.css](styles/theme.css) 提供基础 Token→语义映射。

### 约定

- 组件样式**优先使用语义变量**（如 `--foreground-primary`、`--background-secondary`、`--state-hover`），不硬编码 hex/rgba。
- 新增或修改色值时仅改 `tokens.json`，运行 `generate-tokens.py`。

## 浅色模式

### 背景层级

浅色模式下页面主背景与表面均为白色，无单独 page/hero 区分：

- **主背景 / 表面**：`background.light` = `surface.light` = `#FFFFFF`，对应 `--background-primary`、`--background-secondary`。

| 分类 | Token | 色值 | 用途 / 语义变量 |

## 使用建议

1. **组件样式**：统一使用语义变量（`--foreground-*`、`--background-*`、`--state-*`、`--code-*` 等），不写死 hex/rgba。
2. **改色流程**：只修改 [tokens/tokens.json](tokens/tokens.json)，运行 `python3 scripts/generate-tokens.py`。
3. **对比度**：主文与主背景需满足 WCAG AA（正文 ≥ 4.5:1，大文本 ≥ 3:1）。
