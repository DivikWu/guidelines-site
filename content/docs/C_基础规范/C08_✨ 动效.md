# ✨ 动效

动效为界面注入生命力，帮助用户理解状态变化、引导注意力、提供操作反馈。YAMI 设计系统定义了统一的动效原则与规范，确保动效的一致性与克制性。

---

## 动效原则

### 有目的

每个动效都应有明确的目的：
- 提供操作反馈
- 引导用户注意力
- 表达状态变化
- 建立空间关系

> [!warning] 避免
> 纯装饰性动效会分散注意力，增加认知负担。

### 自然

动效应模拟真实世界的物理规律：
- 遵循惯性与重力
- 避免线性匀速运动
- 使用缓动曲线（easing）

### 快速

动效应尽可能短，不阻碍用户操作：
- 微交互动效保持简短
- 避免用户等待动效完成
- 可打断或跳过

### 一致

相同类型的交互使用相同的动效模式：
- 所有弹窗使用统一的进入/退出动效
- 页面切换动效保持一致
- 相同层级的元素使用相同的过渡时长

## 动效类型

### 过渡动效（Transition）

用于状态之间的切换：
- 页面进入 / 退出
- 弹窗展开 / 收起
- 列表项展开 / 折叠

### 微交互（Micro-interaction）

用于即时反馈：
- 按钮点击反馈
- 开关切换
- 加载指示器
- 输入验证

### 编排动效（Choreography）

用于复杂场景的协调：
- 多个元素依次进入
- 列表项错位动画
- 引导流程

---

## 缓动曲线

缓动曲线决定动效的节奏感：

| 类型 | 用途 |
|------|------|
| Ease-out | 元素进入视图（快速开始，缓慢结束） |
| Ease-in | 元素退出视图（缓慢开始，快速结束） |
| Ease-in-out | 元素在视图内移动 |
| Linear | 仅用于无限循环动效（如 loading） |

## 使用建议

1. **优先考虑减弱动效设置**：尊重用户的 `prefers-reduced-motion` 系统偏好
2. **移动端性能**：避免复杂动效影响帧率
3. **首屏加载**：首屏内容优先展示，动效可延迟
4. **测试**：在真实设备上测试动效流畅度

---

## 常见问题

**Q：所有交互都需要动效吗？**

A：不是。简单的状态切换可以直接变化，动效应用于需要强调或帮助理解的场景。

**Q：如何处理列表项动效？**

A：列表项可采用错位进入动效，但需控制数量。超过一定数量后应简化或取消动效。
