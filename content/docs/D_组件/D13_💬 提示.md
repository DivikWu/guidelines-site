# 💬 提示

提示（Notification/Toast/Tooltip）用于向用户传递轻量级反馈，不阻断操作，适合成功提示、错误提示、说明补充等场景。

## 组件概述

提示组件不抢占焦点，用户可继续操作，适合快速反馈与补充说明。按使用场景分为 Toast、Message、Tooltip 等类型。

## 组件类型

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| Toast 提示 | 浮层展示，自动消失，无操作按钮 | 操作成功、保存成功等即时反馈 |
| Message 通知 | 可带操作按钮，支持手动关闭 | 需用户知晓并可操作的通知 |
| Tooltip 工具提示 | 悬停或聚焦时显示，跟随触发元素 | 图标说明、截断文字全文 |

## 位置

- **Toast/Message**：顶部、底部、中间，需统一规范
- **Tooltip**：跟随触发元素，可配置上/下/左/右

## 设计要点

- **自动消失**：Toast 一般 2–3 秒自动关闭
- **手动关闭**：Message 可提供关闭按钮
- **图标类型**：成功、错误、警告、信息等可配图标
- **动画效果**：进入与退出需有过渡动画，参考 [[C08_✨ 动效|动效规范]]

## 使用规则

**何时使用**
- 操作结果的轻量反馈（成功、失败、保存等）
- 需要补充说明的图标、缩写
- 非阻塞式通知

**何时不使用**
- 重要确认操作用 [[D12_🪟 弹窗|弹窗]]
- 长说明用帮助文档或内联展开
