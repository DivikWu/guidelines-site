# 📱 应用图标规范

应用图标是用户与产品的第一视觉接触点，在 App Store、桌面、系统设置等多个场景中代表品牌形象。本规范定义了 YAMI 应用图标的设计标准与使用规则，确保在所有平台上保持一致的品牌识别度。

---

## 设计原则

### 简洁易识别

- 图标应在任意尺寸下都能被快速识别
- 避免过于复杂的细节，优先保证视觉清晰度
- 核心图形应占据足够的视觉权重

### 与品牌一致

- 图标需要传达 YAMI 品牌的核心调性
- 使用品牌规定的颜色体系
- 保持与其他品牌资产的视觉协调性

### 跨平台适配

- 同一图标需适配不同平台的设计语言（iOS / Android / Web / Desktop）
- 在保持核心识别元素的前提下，遵循各平台的图标设计规范

---

## 图标网格

### 设计画布

| 属性 | 规格 |
|------|------|
| 画布尺寸 | 1024 × 1024 px |
| 安全区域 | 距边缘 64 px |
| 内容区域 | 896 × 896 px |
| 圆角参考 | 根据平台要求自动裁切 |

### 安全区域

- 核心图形应完全位于安全区域内
- 边缘区域可能因平台圆角裁切而被遮挡
- 重要视觉元素不得靠近边缘

> [!warning] 注意
> iOS 系统会自动为图标添加圆角遮罩，设计时需预留足够的安全边距

### 视觉居中

- 图标应基于视觉重心居中，而非几何居中
- 不对称图形需根据视觉平衡微调位置
- 确保图标在各种背景下都显得稳定

## 设计风格

### 图形语言

- **几何化**：优先使用简洁的几何形状构建图标
- **扁平化**：避免过度的立体效果和质感模拟
- **现代感**：保持简约、干净、专业的视觉风格

### 光影处理

- 允许使用轻微的渐变增加层次感
- 避免使用强烈的投影或立体效果
- 光源方向保持一致（建议：左上方 45°）

### 细节控制

| 尺寸范围 | 细节建议 |
|---------|---------|
| ≥ 512px | 可保留完整细节 |
| 128–512px | 适当简化次要细节 |
| < 128px | 仅保留核心识别元素 |

> [!tip] 建议
> 为极小尺寸（如 16px favicon）单独设计简化版本，确保可识别性

## 颜色使用

### 主色应用

- 应用图标应以品牌主色为核心色调
- 具体颜色值请参考 [[品牌色彩策略|品牌色彩策略]]

### 色彩限制

- 单个图标建议使用 2–4 种颜色
- 避免使用过于鲜艳或刺眼的配色
- 渐变方向应保持全系列产品的一致性

### 背景色

- 图标背景应为实色或简单渐变
- 避免使用透明背景（除非平台特别要求）
- 背景色需与前景元素形成足够对比

---

## 禁止事项

> [!danger] 以下用法是不允许的

### 图形修改

- ❌ 拉伸、压缩或扭曲图标比例
- ❌ 旋转或翻转图标
- ❌ 裁切或遮挡图标的任何部分
- ❌ 添加未经授权的装饰元素

### 颜色修改

- ❌ 使用非品牌规定的颜色
- ❌ 随意更改图标的色彩搭配
- ❌ 将图标转为黑白或灰度（除非有特定用途）

### 环境使用

- ❌ 在复杂或干扰性强的背景上使用
- ❌ 在对比度不足的背景上使用
- ❌ 与其他品牌图标组合使用

## 导出规范

### 文件格式

| 格式 | 用途 | 说明 |
|------|------|------|
| PNG | 通用格式 | 支持透明度，适合大多数场景 |
| SVG | Web / 矢量 | 可缩放，适合响应式场景 |
| ICO | Windows | 包含多尺寸 |
| ICNS | macOS | Apple 专用格式 |
| PDF | iOS 资产目录 | 矢量格式，自动生成多尺寸 |

### 命名规范

```
[产品名]_icon_[平台]_[尺寸].[格式]
```

示例：
- `yami_icon_ios_1024.png`
- `yami_icon_android_512.png`
- `yami_icon_web_favicon.ico`

### 色彩模式

| 平台 | 色彩模式 | 色彩空间 |
|------|---------|---------|
| iOS / macOS | RGB | Display P3 |
| Android | RGB | sRGB |
| Web | RGB | sRGB |
| 印刷 | CMYK | - |
