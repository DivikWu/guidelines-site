# ❓ 常见问题

本页面汇总了在使用 YAMI 设计系统过程中最常见的问题，作为日常查阅参考。

## 什么时候必须使用设计系统？

- 只要是正式进入开发的设计方案，都必须使用设计系统中的组件、样式和规范。
- 探索性方案、概念验证可以灵活处理，但在落地前需回归系统。

## 当发现现有组件不太满足需求，怎么办？

- 提交新需求或改进建议
- 基于现有组件做临时方案，并同步给设计系统维护人

## 可以自己新建颜色 / 字体 / 间距吗？

不建议。所有颜色、字体和间距应来自系统定义的 Token / Style。如果现有 Token 不适用，应先讨论是否需要扩展系统。

## 什么时候需要关注更新日志？

在以下场景中，建议查看更新日志：

- 开始新的页面或功能设计
- 使用到近期更新过的组件
- 同步 Figma 组件库之前

## 设计系统和开发规范不一致怎么办？

以最新设计系统规范为准，并及时同步给开发同学。如发现规范本身有问题，应尽快反馈并推动修正。
